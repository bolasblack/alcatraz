## [0.2.0] - 2026-02-24

### Added

- Support fetching config presets from git repos via `alca init git+<url>` ([`30ae5f0`](https://github.com/bolasblack/alcatraz/commit/30ae5f0))

  ```bash
  # Fetch presets from a repo
  alca init git+https://github.com/myorg/alca-presets

  # Fetch from a subdirectory, pinned to a specific commit
  alca init git+https://github.com/myorg/alca-presets#a1b2c3d:backend
  ```

  An interactive prompt lets you pick which `.alca.*.toml` files to download. Each downloaded file includes a source comment for traceability. See [Presets](../config/presets.md).

- Support refreshing managed preset files via `alca init --update` ([`30ae5f0`](https://github.com/bolasblack/alcatraz/commit/30ae5f0))

  Re-fetches all preset files that have a source comment, updating them to the latest version from their origin repos. See [Presets](../config/presets.md).

- Add `${alca:HOST_IP}` token for lan-access rules ([`d6a15d9`](https://github.com/bolasblack/alcatraz/commit/d6a15d9))

  Resolves to the container host's gateway IP at runtime, so firewall rules don't need hardcoded addresses:

  ```toml
  [network]
  lan-access = ["*://${alca:HOST_IP}:3000", "*://${alca:HOST_IP}:8080"]
  ```

  See [Network](../config/network.md).

- Support `${VAR}` environment variable expansion in mount source paths ([`e004664`](https://github.com/bolasblack/alcatraz/commit/e004664))

  ```toml
  mounts = [
    "${HOME}/.ssh:/root/.ssh:ro",
    "${PROJECT_ROOT}/data:/app/data",
  ]
  ```

  Undefined variables are treated as errors — no silent empty-string substitution. See [Config Fields](../config/fields.md).

- Support `${VAR}` environment variable expansion in `extends`/`includes` paths ([`d7f0e05`](https://github.com/bolasblack/alcatraz/commit/d7f0e05))

  ```toml
  extends = ["${HOME}/.alca/base.toml"]
  includes = ["${ALCA_CONF_DIR}/overrides/*.toml"]
  ```

  Variables are expanded before relative path resolution and glob matching. See [Extends & Includes](../config/extends-includes.md).

- Include two built-in preset files (`.alca.claude.toml`, `.alca.nmem.toml`) and add `llms.txt` comment to generated `.alca.toml` ([`332cb83`](https://github.com/bolasblack/alcatraz/commit/332cb83))
